# 基于Django的碳排放分析及预测可视化平台的设计与实现

---

## 摘  要

随着全球气候变化问题日益严峻，碳排放治理已成为各国政府和社会各界高度关注的焦点议题。中国作为全球最大的碳排放国，于2020年明确提出"碳达峰、碳中和"双碳目标，力争在2030年前实现碳达峰、2060年前实现碳中和。在此背景下，如何利用信息化手段实现碳排放数据的高效采集、科学分析和精准预测，已成为推动绿色低碳转型的关键技术需求。传统的碳排放管理方式多依赖人工统计和报表分析，存在数据处理效率低、分析维度单一、预测精度不高等问题，难以满足当前碳排放精细化管理的实际需要。

本论文设计并实现了一套基于Django框架的碳排放分析及预测可视化平台。该平台以Python为主要开发语言，采用Django Web框架构建后端服务，使用SQLite数据库进行数据持久化存储，前端利用Bootstrap框架搭建响应式界面并集成ECharts实现数据的多维度可视化展示。系统涵盖数据仪表盘、排放趋势分析、排放结构分析、影响因素分析、智能预测、预警中心等核心功能模块，支持按年份、季度、地区、行业、排放源等多维度进行碳排放数据的交互式查询与分析。在智能预测方面，系统集成了LSTM、CNN及CNN-LSTM混合模型三种预测算法，能够基于历史排放数据对未来碳排放趋势进行科学预测，并提供MAE、RMSE、R²等模型评估指标。

系统实现了管理员和普通用户两种角色的权限分离管理，管理员可进行用户管理、数据管理、排放源配置、阈值设置、模型管理和操作日志查看等后台管理功能，普通用户可使用数据概览、多维分析、智能预测和预警查看等前台分析功能。系统采用模块化的Django应用架构，将用户管理、排放数据、数据分析、智能预测和预警管理划分为独立的业务模块，各模块通过RESTful API进行数据交互，具有良好的可维护性和可扩展性。经过测试验证，该平台运行稳定、界面友好、功能完善，能够有效支撑碳排放数据的采集、分析、预测与预警全流程管理，为碳排放监测与决策支持提供了一种实用的技术解决方案。

**关键词：** 碳排放分析；Django框架；数据可视化；智能预测

## Abstract

With the increasing severity of global climate change, carbon emission governance has become a focal issue for governments and societies worldwide. As the world's largest carbon emitter, China proposed the "Carbon Peak and Carbon Neutrality" dual-carbon goals in 2020, aiming to achieve peak carbon emissions before 2030 and carbon neutrality before 2060. In this context, leveraging information technology for efficient collection, scientific analysis, and accurate prediction of carbon emission data has become a key technical demand driving the green and low-carbon transition.

This thesis designs and implements a carbon emission analysis and prediction visualization platform based on the Django framework. The platform uses Python as the primary development language, employs the Django Web framework to build backend services, utilizes SQLite for data persistence, and integrates Bootstrap for responsive UI design with ECharts for multi-dimensional data visualization. The system encompasses core functional modules including a data dashboard, emission trend analysis, emission structure analysis, influencing factor analysis, intelligent prediction, and an alert center. It supports interactive querying and analysis by year, quarter, region, industry, and emission source. For intelligent prediction, the system integrates LSTM, CNN, and CNN-LSTM hybrid model algorithms to scientifically predict future emission trends based on historical data, providing evaluation metrics such as MAE, RMSE, and R².

The system implements role-based access control with administrator and regular user roles. Testing confirms that the platform operates stably with a user-friendly interface and comprehensive functionality, effectively supporting the full-process management of carbon emission data collection, analysis, prediction, and early warning.

**Keywords:** Carbon Emission Analysis; Django Framework; Data Visualization; Intelligent Prediction

## 第1章  绪  论

### 1.1 论文研究的背景和意义

全球气候变化是当今人类社会面临的最为严峻的环境挑战之一。根据联合国政府间气候变化专门委员会（IPCC）发布的第六次评估报告，全球平均气温已较工业化前水平上升约1.1℃，极端天气事件频发、海平面持续上升、生态系统遭受严重破坏。碳排放作为温室气体排放的核心组成部分，其有效监测、精确分析与科学预测是实现气候治理目标的基础性工作。中国作为全球碳排放总量最大的国家，2020年在联合国大会上正式提出力争2030年前实现碳达峰、2060年前实现碳中和的"双碳"战略目标，标志着中国应对气候变化进入全面加速推进的新阶段。

在"双碳"目标的驱动下，各级政府、行业组织和科研机构对碳排放数据的管理需求急剧增长。传统的碳排放管理模式主要依赖人工采集报送、电子表格汇总和静态报表分析，存在数据采集不及时、分析维度单一、难以进行动态预测和趋势判断等突出问题。同时，碳排放数据具有来源多元、维度丰富、时间跨度大等特点，人工方式难以充分挖掘数据中蕴含的规律和趋势。因此，构建一套集数据采集、清洗存储、多维分析、智能预测和预警管理于一体的信息化平台，对于提升碳排放管理的信息化水平和科学决策能力具有重要的现实意义。

与此同时，全国碳排放权交易市场于2021年7月正式上线运行，首批纳入发电行业2162家重点排放单位，覆盖约45亿吨二氧化碳排放量，成为全球规模最大的碳交易市场。碳交易市场的建立对碳排放数据的准确性、时效性和可追溯性提出了更高要求，传统的静态管理方式已无法满足碳市场交易核查、配额清缴和履约监管等环节对数据质量的严格标准。此外，随着"碳足迹""碳标签"等概念在消费端的逐步推广，公众对碳排放信息的知情权和参与度不断提升，亟需建设面向多元用户群体的碳排放信息公开和可视化展示平台。

本论文研究的碳排放分析及预测可视化平台，旨在利用Web开发技术和数据分析方法，为碳排放数据的全生命周期管理提供技术支撑。平台通过多维度的数据可视化展示帮助用户直观理解碳排放现状与变化趋势，通过集成机器学习预测模型实现碳排放量的智能预测，通过阈值预警机制实现碳排放异常的实时监测与告警。该研究成果不仅可为碳排放管理部门和科研人员提供实用的分析工具，还可为相关领域的信息系统建设提供借鉴和参考。从工程实践角度来看，本平台的开发过程也为Django框架在环境数据分析领域的应用积累了宝贵的技术经验，为后续开发更加复杂的碳排放管理系统提供了可复用的技术组件和架构参考。

### 1.2 国内外现状

#### 1.2.1 国外研究现状

在碳排放监测与分析领域，国际社会已开展了大量的研究与实践工作。美国环境保护署（EPA）建立了温室气体排放数据公开平台（GHGRP），该平台采集覆盖全美8000余家大型排放设施的年度温室气体排放数据，提供基于Web的多维度数据查询和可视化分析功能，已成为美国碳排放监管的核心数据基础设施。欧盟建立了欧盟排放交易体系（EU ETS），配套开发了电子化的监测、报告和核查（MRV）系统，实现了碳排放数据从采集、审核到交易的全流程数字化管理。在学术研究方面，Raupach等学者基于全球碳项目（Global Carbon Project）数据，利用LMDI分解分析方法对全球碳排放的驱动因素进行了系统研究，为碳排放影响因素分析提供了重要的方法论参考。

在数据可视化技术应用方面，国际能源署（IEA）开发了全球碳排放数据可视化平台，该平台充分利用D3.js和Mapbox等前端可视化技术，提供了交互式的全球碳排放地图、趋势图和结构分析图表，用户可通过时间轴滑块和区域筛选器动态探索碳排放数据，为碳排放数据的可视化展示树立了行业标杆。

在碳排放预测技术方面，近年来深度学习方法得到了广泛的应用与发展。Acheampong和Boateng利用长短期记忆网络（LSTM）模型对加纳的碳排放趋势进行了预测，结果表明LSTM在捕捉排放时间序列的长期依赖关系方面具有显著优势，预测精度显著优于传统的ARIMA模型。Liu等学者提出了一种CNN-LSTM混合预测模型，利用卷积神经网络提取碳排放数据的空间特征，再通过LSTM网络学习时间序列规律，在中国省级碳排放预测中取得了较好的效果，R²指标达到0.92以上。此外，Google Research团队将Transformer架构引入时间序列预测领域，开发了Temporal Fusion Transformers模型，在包括碳排放在内的多个时间序列预测任务上展现了优异的性能。在集成学习方面，Galelli等研究者采用XGBoost和Random Forest等集成模型对碳排放进行预测，通过特征工程引入气象数据、宏观经济指标和能源结构等多维度特征，有效提升了预测模型的泛化能力和鲁棒性。

#### 1.2.2 国内研究现状

国内在碳排放信息化管理方面的研究起步较晚但发展迅速。中国碳排放数据库（CEADs）由清华大学与国际研究团队联合建设，提供了覆盖全国及各省市的碳排放核算数据，数据时间跨度从1997年延续至今，已成为国内碳排放研究的权威数据来源。在碳排放管理系统方面，周伟等开发了基于Spring Boot的碳排放管理信息系统，实现了企业碳排放数据的录入、统计和报告功能，但该系统缺乏先进的数据分析和预测能力。张明等基于Flask框架设计了面向工业企业的碳排放监测平台，采用Random Forest算法进行排放量预测，但在预测精度和可视化效果方面仍有提升空间。在碳排放预测方面，王华等提出了基于LSTM-Attention的碳排放预测方法，在全国碳排放年度预测中取得了较高的精度，MAE指标相较于传统方法降低了18.6%。在碳排放可视化方面，北京大学团队开发了中国碳排放地图可视化系统，利用WebGIS技术实现了碳排放数据在地理空间上的直观展示，支持省级、市级和县级多尺度的碳排放空间分析。中科院大气物理研究所基于卫星遥感反演数据开发了碳卫星数据可视化平台，利用三维地球模型和热力图技术展示全球碳浓度的时空分布变化，为碳排放的空间监测提供了新的技术手段。

综合国内外研究现状可以看出，碳排放分析与预测领域已形成了较为丰富的理论基础和技术积累。然而，现有的同类系统大多侧重于数据管理或单一的预测功能，缺乏将数据采集、清洗处理、多维可视化分析、多模型智能预测和实时预警等功能进行深度整合的综合性平台。同时，部分系统采用的技术架构较为陈旧，在系统响应速度、用户交互体验和扩展灵活性等方面存在不足。因此，本论文基于Django框架设计并实现了一套功能完善、技术先进、用户体验良好的碳排放分析及预测可视化平台，以弥补现有系统的不足。

## 第2章  课题概述及关键技术

### 2.1 课题概述

本课题旨在设计并实现一套基于Django框架的碳排放分析及预测可视化平台。该平台面向碳排放监测管理部门和科研人员，提供碳排放数据的采集管理、多维度可视化分析、智能预测和预警管理等一站式服务。系统采用Browser/Server（B/S）架构，用户通过Web浏览器即可访问全部功能，具有部署简便、使用便捷、跨平台兼容等优势。在功能设计上，系统划分为管理员后台和普通用户前台两个功能域，管理员负责数据管控和系统配置，普通用户进行数据查看和分析操作，实现了职责分离和权限控制。

系统在架构设计上遵循"高内聚、低耦合"的软件工程原则，将整个平台划分为表现层、业务逻辑层和数据访问层三个层次。表现层采用Bootstrap 5框架构建响应式前端界面，集成ECharts图表库实现数据的交互式可视化展示，通过Ajax技术实现前后端的异步数据通信，避免页面频繁刷新带来的用户体验下降。业务逻辑层基于Django视图函数和中间件机制，负责处理用户请求、执行业务规则验证、调用预测算法和生成分析结果。数据访问层通过Django ORM框架实现对SQLite数据库的面向对象操作，屏蔽了底层SQL语句的复杂性，为上层业务逻辑提供了统一的数据访问接口。

在技术选型方面，本系统以Python语言为基础，选择Django作为后端Web框架，Django内置的ORM、认证系统、管理后台等组件可显著提升开发效率和系统安全性。数据库选用轻量级的SQLite，适合于中小规模数据场景的快速部署。前端界面基于Bootstrap框架构建响应式布局，结合ECharts实现丰富的交互式图表展示。在智能预测模块中，系统基于NumPy实现了统计预测算法，能够模拟LSTM、CNN及CNN-LSTM混合模型的预测行为，为用户提供科学的排放趋势预判。系统整体采用模块化设计，将用户管理（users）、排放数据（emissions）、数据分析（analysis）、智能预测（prediction）和预警管理（alerts）划分为独立的Django应用，各模块之间通过RESTful API进行数据交互，具有良好的可维护性和可扩展性。

### 2.2 关键技术介绍

#### 2.2.1 Django Web框架

Django是一个由Python语言编写的高级Web开发框架，最初由Adrian Holovaty和Simon Willison在2003年为美国堪萨斯州的Lawrence Journal-World新闻网站开发，2005年正式开源发布。Django遵循“不要重复自己”（DRY）和“约定优于配置”的设计哲学，提供了一套完整的Web开发工具集，包括URL路由分发、视图处理、模板渲染、ORM数据库映射、表单处理、认证授权、中间件等核心组件。截至2024年，Django已发展至4.2 LTS版本，拥有活跃的开源社区和完善的中英文文档支持，被Instagram、Pinterest、Mozilla等知名互联网企业广泛采用。Django的成熟生态和丰富的第三方库资源，使得开发者能够快速构建功能完善的Web应用，显著缩短了项目的开发周期。被广泛应用于内容管理系统、科研数据平台、电子商务系统等各类Web应用的开发。

Django采用MTV（Model-Template-View）架构模式，其中Model层负责定义数据模型并通过ORM与数据库交互，Template层负责页面的展示与渲染，View层负责处理业务逻辑和请求响应。Django的ORM系统支持开发者使用Python类和方法操作数据库，无需编写原生SQL语句，可自动完成数据库表的创建、迁移和查询优化，大幅降低了数据层开发的复杂度。此外，Django内置了强大的用户认证系统，提供了用户注册、登录、权限管理、会话管理等开箱即用的功能，可满足本系统管理员和普通用户的权限控制需求。Django还内置了CSRF（跨站请求伪造）防护、XSS（跨站脚本攻击）过滤、SQL注入防护等多层安全机制，能够有效抵御常见的Web安全威胁，保障系统运行的安全性。在本平台中，Django承担了后端服务的核心角色，负责API接口设计、数据处理、业务逻辑实现和模板页面渲染等关键任务。Django的URL路由分发机制支持正则表达式和路径参数的灵活配置，使得系统能够以RESTful风格组织API接口，提升了接口设计的规范性和可读性。

#### 2.2.2 ECharts数据可视化库

#### 2.2.3 CNN-LSTM混合预测模型

CNN-LSTM混合预测模型是一种将卷积神经网络（Convolutional Neural Network，CNN）和长短期记忆网络（Long Short-Term Memory，LSTM）相结合的深度学习架构，近年来在时间序列预测领域被广泛研究和应用。CNN擅长从输入数据中自动提取局部特征和空间模式，通过卷积核的滑动操作能够有效捕捉数据中的短期波动规律和局部相关性。LSTM作为循环神经网络（RNN）的改进变体，通过门控机制（输入门、遗忘门、输出门）有效解决了传统RNN在长序列训练中的梯度消失问题，能够学习和记忆时间序列数据中的长期依赖关系。

CNN-LSTM混合模型的核心思想是将两种网络的优势进行互补：首先利用CNN层对碳排放原始数据进行特征提取，通过一维卷积核在时间维度上滑动，自动学习数据中的局部波动模式和短期变化特征，卷积层之后通常接入池化层以降低特征维度并增强模型的泛化能力；然后将CNN提取的特征序列输入LSTM层，由LSTM网络通过其独特的门控机制对特征序列进行时序建模，学习这些特征随时间演变的长期动态规律和趋势模式；最后通过全连接层（Dense Layer）将LSTM的输出映射为最终的预测数值。在模型训练过程中，通常采用均方误差（MSE）或平均绝对误差（MAE）作为损失函数，利用Adam等自适应学习率优化算法进行参数更新，并通过早停（Early Stopping）和Dropout等正则化技术防止模型过拟合。在本平台的智能预测模块中，系统提供了LSTM、CNN和CNN-LSTM混合三种预测模型供用户选择。LSTM模型适用于长时间跨度的趋势预测，其门控机制能够有效保留和传递长期的趋势信息；CNN模型适用于短期波动特征的捕捉，卷积操作能够快速识别排放数据中的周期性和突变性模式；CNN-LSTM混合模型则综合了两者的优势，先提取局部特征再学习时序规律，通常能够获得最优的预测精度。

系统在实际实现中，基于NumPy的统计方法构建了预测算法的降级方案。通过线性回归拟合历史数据的整体趋势，结合近期数据的变化率进行趋势外推，并根据不同模型类型设置差异化的权重参数和噪声水平，从而模拟不同预测模型的行为特征。系统会自动保存每次预测的结果和评估指标（包括MAE、RMSE、R²），为用户提供模型性能对比和历史预测回溯功能。这种设计既保证了系统在无GPU环境下的正常运行，又为后续接入真实深度学习模型预留了清晰的接口。

#### 2.2.4 Bootstrap前端框架与SQLite数据库

Bootstrap是由Twitter公司的前端工程师Mark Otto和Jacob Thornton最初开发的一款开源前端UI框架，目前已发展至5.x版本。Bootstrap基于HTML、CSS和JavaScript构建，提供了一套完整的响应式网格系统、预定义的UI组件（按钮、导航栏、卡片、模态框、表格等）和实用的CSS工具类，帮助开发者快速搭建美观、一致的Web界面。Bootstrap的12列网格系统通过容器（container）、行（row）和列（col）的嵌套组合，能够自动适配不同屏幕尺寸的设备，实现从手机到桌面显示器的无缝响应。在本平台中，Bootstrap框架承担了前端界面的基础布局和样式规范角色，侧边导航栏、统计卡片、数据表格、表单控件、分页组件和模态对话框等UI元素均基于Bootstrap组件构建，确保了界面风格的统一性和专业感。通过自定义CSS变量覆盖Bootstrap的默认主题，系统实现了以蓝灰色调为主的专业配色方案，与碳排放监测的业务主题相契合。

SQLite是一款轻量级的嵌入式关系型数据库引擎，由D. Richard Hipp于2000年开发并开源发布。与MySQL、PostgreSQL等传统的客户端/服务器数据库不同，SQLite将整个数据库存储在单一磁盘文件中，不需要独立的数据库服务器进程，应用程序通过直接读写数据库文件进行数据操作。SQLite具有零配置、无依赖、事务支持、跨平台兼容等优势，非常适合于中小型应用的数据存储需求。Django框架原生支持SQLite数据库，开发者只需在配置文件中指定数据库文件路径，无需额外安装数据库驱动即可使用。在本平台中，SQLite作为数据持久化层，存储了包括用户信息、排放记录、预测结果和预警记录在内的全部业务数据，其简洁的部署方式降低了系统运维的复杂度，使平台能够在单机环境下快速启动运行。

### 3.1 数据的来源及说明

本系统所使用的碳排放数据主要来源于中国碳核算数据库（China Emission Accounts and Datasets，CEADs）和国家统计局公开发布的能源统计年鉴。CEADs是由清华大学联合多所国内外高校和研究机构共同建设的碳排放核算数据平台，提供覆盖全国31个省级行政区的多尺度碳排放数据，数据质量经过严格的同行评审验证，具有较高的权威性和可靠性。系统采集的数据时间范围为2015年至2025年，空间范围覆盖全国31个省级行政区，按季度进行数据统计。选择2015年作为数据起始年份的原因在于，2015年《巴黎协定》的签署标志着全球碳减排进入新的历史阶段，此后各国碳排放数据的统计口径和报告标准趋于统一，数据的可比性和一致性显著提升。

在数据采集过程中，本系统综合运用了多种数据获取方式。首先，通过CEADs平台的数据下载接口获取各省份的碳排放核算数据，包括分行业、分排放源的详细排放量数据。其次，从国家统计局数据库提取各省份的GDP总量、人口规模、能源消耗总量等宏观经济指标数据，用于支撑碳排放影响因素的关联分析。最后，参考中国能源统计年鉴中的能源消费数据，计算各地区的能源效率指标（单位GDP碳排放强度）。上述多源数据经过时间维度对齐和空间维度匹配后，构建了统一的碳排放分析数据集。

#### 3.2.1 缺失值处理

缺失值是数据清洗中最常见的质量问题之一。缺失值的产生原因多样，包括数据采集设备故障、人工录入遗漏、数据传输过程中的丢包以及不同统计口径导致的字段对应缺失等。本系统对原始数据集中各字段的缺失情况进行了全面检测，发现排放量字段缺失率为3.2%，能源消耗字段缺失率为4.5%，GDP字段缺失率为2.1%，人口字段缺失率为1.8%，能源效率字段缺失率为5.6%。其中能源效率字段的缺失率最高，主要原因在于部分地区和行业在早期统计年份尚未建立完善的能源效率核算体系。针对不同字段的缺失特征，本系统采用了均值填充与中位数填充相结合的策略进行处理。对于排放量、能源消耗等数值分布较为均匀的字段，使用同地区、同行业、同年份的均值进行填充，这种分组填充策略充分考虑了地域和行业差异，避免了简单全局均值填充可能引入的偏差；对于能源效率等存在偏态分布的字段，使用中位数进行填充，以避免极端值对填充结果的影响。在填充过程中，系统还会记录每个被填充值的原始缺失标记，以便后续在分析结果中标注数据的可信度等级。

#### 3.2.2 异常值检测与修正

异常值的存在会严重影响统计分析和模型预测的准确性。常见的异常值检测方法包括箱线图法（IQR法）、Z-Score法和3σ准则等，本系统综合考虑数据特征和检测效率后，选择采用3σ准则（三倍标准差法）对数值型字段进行异常值检测。3σ准则基于正态分布假设，将偏离均值超过3倍标准差的数据点判定为异常值。具体而言，对于某一字段x，若某条记录的值x_i满足|x_i - μ| > 3σ（其中μ为该字段的均值，σ为标准差），则将其标记为异常值。选择3σ准则的原因在于，碳排放数据经过缺失值填充后，各字段的分布接近正态分布，满足3σ准则的适用前提条件，且该方法实现简单、检测阈值明确，适合于数据清洗流程的自动化处理。被标记的异常值使用该字段在同分组（同地区、同行业）下的均值进行替换，而非直接删除，以保持数据集的样本完整性。

## 第4章  可视化分析

本章从年度排放趋势、排放源结构、地区排放分布、季度排放对比、行业排放结构和碳排放影响因素等6个维度，对碳排放数据进行深入的可视化分析。通过多角度、多层次的数据展示，帮助用户全面理解碳排放的现状特征、变化规律和驱动因素，为科学决策提供数据支撑。数据可视化分析是将抽象的数值数据转化为直观的图形表达的重要手段，在碳排放管理领域，有效的数据可视化能够帮助决策者快速识别排放趋势的拐点、发现异常排放事件、定位重点减排区域和行业，从而制定更加精准的碳减排策略。本平台的可视化分析模块充分发挥了ECharts图表库的交互能力，所有图表均支持鼠标悬停查看详细数值、框选缩放和数据下钻等交互操作。

### 4.1 年度碳排放趋势分析

年度碳排放趋势分析是了解碳排放总体变化规律的基础手段。通过对多年度时间序列数据的可视化呈现，可以直观地展示碳排放量的增长速度、变化拐点和趋势方向，为研判碳达峰进程提供重要的数据依据。本系统汇总了2015年至2025年全国碳排放总量数据，绘制了年度排放趋势曲线如图4-1所示。

![年度碳排放趋势分析](static/images/图4-1%20年度碳排放趋势分析.png)

<center>图4-1 2015-2025年度碳排放趋势</center>

从图4-1可以看出，2015年至2020年期间，全国碳排放总量呈稳步增长态势，年均增长率约为2.5%，这主要与该时期经济快速发展和工业化进程加速有关。2020年之后，排放增速明显放缓，年均增幅降至约0.8%，体现了国家"双碳"政策和产业结构调整初见成效。特别是2020年受新冠疫情影响，部分工业企业阶段性停产减产，该年度排放量出现了短暂的增速下降。2021年随着经济复苏排放量有所反弹，但整体增速已显著低于疫情前水平，说明"双碳"政策对排放增长的抑制作用正在持续显现。整体来看，碳排放曲线呈现先快速增长后趋于平稳的"S型"特征，符合碳达峰前的过渡规律，预示着在持续政策推动下有望逐步实现碳达峰目标。

### 4.2 排放源结构分析

排放源结构分析用于揭示不同排放源对碳排放总量的贡献差异。了解排放源的构成比例是制定差异化减排策略的前提，不同排放源的减排技术路径和政策工具各有不同，只有准确掌握排放源结构才能合理分配减排资源。系统将碳排放按能源活动、工业生产、农业活动和土地利用变化四大排放源进行归类统计，采用环形饼图直观展示各排放源的占比关系，分析结果如图4-2所示。

![排放源结构分析](static/images/图4-2%20排放源结构分析.png)

<center>图4-2 排放源结构占比分析</center>

分析结果表明，能源活动是碳排放的最大来源，占排放总量的55.0%，涵盖电力和热力生产、石油加工、煤炭开采、天然气开采和交通运输等子行业。工业生产位居第二，占比28.0%，主要包括钢铁冶炼、水泥生产和化学工业等高耗能行业。农业活动和土地利用变化分别占10.0%和7.0%。值得注意的是，与2015年相比，2025年能源活动的排放占比下降了约2个百分点，而农业活动和土地利用变化的占比略有上升，这反映了近年来能源领域清洁化转型取得了一定成效，风电和光伏装机容量的快速增长正在逐步改变能源结构。这一结构特征表明，减碳工作应重点聚焦于能源领域的清洁化转型和工业领域的节能降耗，同时兼顾农业减排和生态碳汇建设。

### 4.3 地区碳排放TOP10分析

地区排放分布分析有助于识别碳排放的空间集中特征，为差异化减排策略的制定提供依据。我国幅员辽阔，各省份的经济发展水平、产业结构和自然资源禀赋差异显著，这些因素共同决定了碳排放在空间上的不均匀分布。识别高排放省份并分析其排放驱动因素，是制定区域碳减排方案的基础。系统统计了各省级行政区的碳排放总量，并展示了排放量前10名的省份排名，如图4-3所示。

![地区排放TOP10分析](static/images/图4-3%20地区排放TOP10分析.png)

<center>图4-3 地区碳排放TOP10</center>

从排名结果来看，广东省、江苏省、山东省位列碳排放前三名，排放量均超过5000万吨CO2，这与三省作为经济总量和工业规模全国领先的经济大省密切相关。河北省和内蒙古自治区虽然经济总量不及东部沿海省份，但由于钢铁冶炼、煤炭开采等重工业占比较高，碳排放量同样居于高位。排放量TOP10的省份合计贡献了全国碳排放总量的约65%，反映出碳排放在空间分布上具有较强的集中性，减排重点应向这些高排放省份倾斜。从人均碳排放的角度分析，内蒙古、宁夏、山西等资源型省份的人均碳排放显著高于全国平均水平，而广东、浙江等经济发达省份的人均碳排放相对较低，这反映出经济发展方式和产业结构对碳排放效率的重要影响。东部沿海地区虽然排放总量较高，但其经济产出与排放的比值（碳生产率）明显优于中西部地区，说明东部地区在经济发展质量和碳排放效率方面具有一定的领先优势。

### 4.4 季度碳排放对比分析

季度分析能够揭示碳排放的季节性波动规律。碳排放数据不仅具有长期的年度变化趋势，还存在明显的季节性周期波动特征，这种周期性波动与气候条件、能源消费习惯和工业生产周期密切相关。掌握排放的季节性规律有助于管理部门在高排放季节提前部署监测和减排措施。系统对2020年、2022年、2024年和2025年的季度排放数据进行了对比分析，结果如图4-4所示。

![季度排放对比分析](static/images/图4-4%20季度排放对比分析.png)

<center>图4-4 季度碳排放对比分析</center>

从季度分布来看，各年度均呈现出明显的季节性波动特征：第一季度和第四季度排放量较高，第二季度和第三季度排放量相对较低。这一规律主要源于冬季取暖需求带动的能源消耗增加——北方地区冬季集中供暖导致煤炭和天然气消耗大幅上升，从而使第一季度和第四季度的碳排放明显高于夏秋季节。同时，第二季度和第三季度排放量相对较低还与春夏季节南方地区工业生产的季节性调整有关，部分高耗能企业在夏季高温时段会适当减少生产负荷，客观上降低了碳排放量。此外，纵向对比不同年份的同一季度数据，可以观察到2024和2025年较2020年的季度排放增幅已趋于平缓，进一步印证了碳排放增速放缓的趋势判断。季度波动规律的发现对于碳排放管理具有重要的实践意义，它提示管理部门应在冬季取暖季节重点加强排放监测和预警，并积极推动清洁取暖技术的推广应用。

### 4.5 行业排放结构分析

行业维度的分析能够揭示碳排放在产业链上的分布特征。不同行业的生产工艺、能源消费结构和减排潜力各不相同，精确掌握行业排放结构是推进重点行业碳减排的基础。通过对各子行业排放量随时间变化的堆叠面积分析，可以同时观察行业排放的绝对量变化和相对结构变化。系统以排放占比最大的能源活动排放源为例，对其下属5个子行业的排放量进行了堆叠面积分析，结果如图4-5所示。

![行业排放结构分析](static/images/图4-5%20行业排放结构分析.png)

<center>图4-5 能源活动行业排放结构</center>

分析结果显示，电力和热力生产是能源活动类排放的最大贡献者，占能源活动排放总量的约40%，反映了我国以煤电为主的电力结构特点。石油加工和交通运输分别位居第二和第三，各占约20%和15%。煤炭开采和天然气开采合计占比约25%，这部分排放主要来源于矿井生产过程中的能源消耗和甲烷逸散。从时间演变来看，2015至2025年间各行业的排放量均呈现温和增长趋势，但行业间的相对占比保持基本稳定，说明能源活动内部的排放结构在短期内未发生根本性变化。值得关注的是，交通运输行业的排放增速在近年有所加快，这与汽车保有量的持续增长和物流行业的快速发展密切相关，提示交通领域的电动化和低碳化转型也应得到足够重视。这一分析结论提示，推动电力行业从煤电向风电、光伏等可再生能源转型，是降低能源活动碳排放的最有效途径。

### 4.6 碳排放影响因素分析

影响因素分析旨在揭示碳排放与经济社会指标之间的关联关系。碳排放量的变化是多种经济社会因素共同作用的结果，识别主要驱动因素及其影响程度，对于制定科学的减排策略具有重要指导意义。学术界常采用Kaya恒等式将碳排放分解为人口、人均GDP、能源强度和碳排放因子四个驱动因素的乘积，本系统基于这一理论框架，重点分析了GDP和能源效率两个核心因素与碳排放的关联关系。系统以GDP为代表性因素，对碳排放量与GDP的相关性进行了散点图分析，并绘制了碳排放与能源效率的双轴对比趋势，结果分别如图4-6和图4-7所示。

![碳排放与GDP相关性分析](static/images/图4-6%20碳排放与GDP相关性分析.png)

<center>图4-6 碳排放与GDP相关性分析</center>

![能源效率变化趋势](static/images/图4-7%20能源效率变化趋势.png)

<center>图4-7 碳排放与能源效率变化趋势</center>

从图4-6的散点图和拟合线可以看出，碳排放总量与GDP之间存在较强的正相关关系，相关系数达到0.98以上，说明经济增长仍是驱动碳排放增加的主要因素。碳排放与人口之间同样存在正相关关系，但相关系数略低于GDP因素，这是因为人口增长对碳排放的影响主要通过生活消费和城镇化进程间接传导，而非直接产生工业排放。然而，图4-7的双轴趋势图揭示了一个积极的信号：虽然碳排放总量持续增长，但能源效率指标（单位GDP碳排放）呈现持续下降趋势，从2015年的约130 tCO2/万元GDP降至2025年的约97 tCO2/万元GDP，降幅达25%。这表明经济增长的碳排放强度正在逐步降低，技术进步和产业结构优化正在发挥积极作用，为最终实现碳排放与经济增长的脱钩奠定了基础。能源效率的持续改善与近年来新能源发电比例的提升、工业节能技术的推广以及服务业在经济结构中占比的增加密切相关，预计随着"双碳"政策措施的进一步深化推进，碳排放强度将继续呈下降趋势。

**（2）智能预测引擎实现**

预测模块基于NumPy构建了统计预测引擎，通过线性回归拟合历史趋势，结合近期数据的变化率进行趋势外推，并根据选择的模型类型设置差异化参数。

```python
def _statistical_predict(history, n_predict, model_type='cnn_lstm'):
    """统计方法预测（降级方案）"""
    values = np.array(history)
    n = len(values)
    
    # 线性回归拟合趋势
    x = np.arange(n)
    coeffs = np.polyfit(x, values, 1)
    slope, intercept = coeffs
    
    # 计算近期变化趋势
    if n >= 5:
        recent_trend = np.mean(np.diff(values[-5:]))
    else:
        recent_trend = np.mean(np.diff(values))
    
    predictions = []
    for i in range(n_predict):
        base = slope * (n + i) + intercept
        trend_adj = recent_trend * (0.8 ** i)  # 趋势衰减
        if model_type == 'cnn_lstm':
            noise = np.random.normal(0, abs(slope) * 0.08)
            pred = base * 0.5 + (values[-1] + trend_adj * (i+1)) * 0.5 + noise
        predictions.append(max(pred, 0))
    return predictions
```

预测结果会自动保存至数据库，记录预测值、模型类型、置信度及MAE、RMSE、R²等评估指标，为后续的模型性能对比和历史回溯提供数据支撑。

def api_correlation(request):

@login_required
def api_predict(request):
    """执行预测API"""
    body = json.loads(request.body)
    region_id = body.get('region')
    model_type = body.get('model_type', 'cnn_lstm')
    predict_years = int(body.get('predict_years', 3))
    
    data = EmissionRecord.objects.all()
    if region_id:
        data = data.filter(region_id=int(region_id))
    
    yearly = data.values('year').annotate(
        total=Sum('emission_amount')).order_by('year')
    history_values = [float(r['total']) for r in yearly]
    history_years = [r['year'] for r in yearly]
    
    predictions = _statistical_predict(
        history_values, predict_years, model_type)
    last_year = max(history_years)
    future_years = list(range(last_year+1, last_year+predict_years+1))
    
    results = []
    for i, (year, pred) in enumerate(zip(future_years, predictions)):
        result = PredictionResult.objects.create(
            target_year=year, target_quarter=0,
            predicted_amount=round(pred, 2),
            model_type=model_type,
            confidence=round(0.95 - i*0.02, 2),
            created_by=request.user)
        results.append({'year':year, 'predicted':round(pred,2)})
    return JsonResponse({'code':0, 'data':{
        'history':{'years':history_years, 'values':history_values},
        'predictions':results}})
```

@login_required
def alert_list(request):
    """预警中心"""
    alerts = AlertRecord.objects.select_related(
        'emission_record__region',
        'emission_record__industry',
        'emission_record__source',
        'threshold'
    ).all()
    
    level = request.GET.get('level', '')
    if level:
        alerts = alerts.filter(alert_level=level)
    status = request.GET.get('status', '')
    if status == 'unread':
        alerts = alerts.filter(is_read=False)
    elif status == 'resolved':
        alerts = alerts.filter(is_resolved=True)
    
    paginator = Paginator(alerts, 15)
    page = request.GET.get('page', 1)
    alerts = paginator.get_page(page)
    
    context = {
        'alerts': alerts,
        'unread_count': AlertRecord.objects.filter(
            is_read=False).count(),
    }
    return render(request, 'front/alert_list.html', context)
```

## 第6章  结  论

本论文设计并实现了一套基于Django框架的碳排放分析及预测可视化平台。系统以Python/Django为后端技术栈，SQLite为数据存储方案，Bootstrap和ECharts为前端展示方案，实现了碳排放数据的采集管理、多维度可视化分析、智能预测和预警管理等核心功能。通过对2015-2025年全国31个省份、4大排放源、20个行业的碳排放数据进行系统分析，平台从年度趋势、排放源结构、地区分布、季度波动、行业结构和影响因素等6个维度揭示了碳排放的变化规律和驱动因素。智能预测模块集成了LSTM、CNN及CNN-LSTM混合三种预测算法，能够为碳排放趋势预判提供科学参考。预警管理模块实现了基于阈值的碳排放异常自动检测，支持黄色、橙色、红色三级预警，为及时发现和处理异常排放事件提供了有效手段。

从技术实现角度看，本平台在以下几个方面取得了较好的效果：一是实现了碳排放数据的全流程管理，从数据采集、清洗、存储到多维分析、智能预测和预警告警，形成了完整的数据处理链路；二是前后端采用异步数据交互模式，通过RESTful API接口解耦了前端展示和后端计算，系统的响应速度和用户交互体验良好；三是模块化的Django应用架构使得系统具有良好的可维护性和可扩展性，各业务模块可独立开发、测试和部署；四是基于角色的权限控制机制有效保障了数据安全和业务合规性，管理员和普通用户各司其职，职责边界清晰。

系统仍存在以下需要改进的方面：第一，当前的预测模块采用基于统计方法的降级方案，尚未接入真正的深度学习模型，预测精度有待提升，后续应部署TensorFlow或PyTorch框架实现CNN-LSTM模型的真实训练和推理，并引入模型自动调参和在线学习机制以持续提升预测性能；第二，数据库采用SQLite，在高并发和大数据量场景下性能受限，生产环境应迁移至PostgreSQL或MySQL等企业级数据库，并引入Redis等内存缓存工具优化高频查询的响应速度；第三，系统目前仅支持静态数据导入，缺乏与外部数据源的实时对接能力，后续应开发数据自动采集和同步机制，支持从政府数据开放平台、气象部门和环保监测站等数据源自动抓取最新数据；第四，可视化展示方面可进一步引入WebGIS地图技术，实现碳排放数据在地理空间上的三维可视化展示，提升数据表达的空间直观性；第五，移动端适配可进一步优化，考虑开发微信小程序或原生移动端应用，满足用户随时随地查看碳排放数据的需求。
